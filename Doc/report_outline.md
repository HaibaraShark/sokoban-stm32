## 二、摘要 (150字)
本项目基于STM32F107VCT6嵌入式平台, 使用ILI9341 TFT液晶屏、GPIO按键和蜂鸣器,
设计实现了一款多关卡推箱子游戏。游戏支持按键/串口双模式操控、步数统计、
撤销回退、过关动画等功能, 包含15个难度递增的关卡。

## 三、系统方案设计

### 3.1 硬件平台
- MCU: STM32F107VCT6 (Cortex-M3, 72MHz, 256KB Flash, 64KB SRAM)
- 显示: ILI9341 3.2寸 TFT LCD (320×240, FSMC 16位并行接口)
- 输入: 4路GPIO按键 (上/下/左/右)
- 音频: PC0蜂鸣器 (软件PWM驱动)
- 通信: USART1 串口 (115200bps, PC键盘扩展)

### 3.2 软件架构
```
应用层: 状态机 (启动画面→菜单→游戏/选关/帮助)
  ├── 菜单: 开始游戏 / 选关 / 帮助
  ├── 游戏: 输入→逻辑→渲染 主循环
  └── 选关: 4×4网格选择界面
逻辑层:
  ├── 地图管理 (双数组分离: ground[] + entities[])
  ├── 移动引擎 (碰撞检测→推动→胜利判定)
  ├── 撤销管理 (环形缓冲区, 50步)
  └── 计分系统 (备份寄存器持久化)
渲染层:
  ├── 瓦片绘制 (6种瓦片, 程序化绘制)
  ├── UI组件 (按钮/菜单/关卡网格)
  └── 动画效果 (启动/过关)
驱动层:
  ├── LCD (FSMC ILI9341)
  ├── 按键 (10ms轮询+去抖+长短按识别)
  ├── 蜂鸣器 (频率/时长控制)
  └── 串口 (中断接收+环形缓冲)
```

### 3.3 关键技术决策
| 问题 | 方案 | 原因 |
|------|------|------|
| RTOS? | 裸机+大循环 | 游戏单线程, RTOS浪费资源 |
| 帧缓冲? | 脏矩形逐瓦片渲染 | 64KB SRAM不够全帧缓冲 |
| 关卡存储? | Flash嵌入 | 15关≈12KB, 256KB Flash充裕 |
| 高分存储? | 备份寄存器(BKP) | 电池供电, 掉电不丢 |
| 横屏/竖屏? | 横屏320×240 | 左侧游戏区, 右侧信息栏 |

## 四、详细设计与实现

### 4.1 地图数据结构
- ground[m×n]: 地格层 (0=空地, 1=墙, 2=目标点)
- entities[m×n]: 实体层 (0=空, 1=玩家, 2=箱子)
- 最大支持20×20, 15关≈12KB

### 4.2 移动算法
```
输入方向(dx,dy) → 计算新位置(nx,ny)
├── 检查是否有墙 → 返回碰撞
├── 检查是否有箱子
│   ├── 箱子前有墙/箱子 → 返回碰撞
│   └── 推动: 箱子移动一格
└── 移动玩家 → 更新地图 → 胜利判定
```

### 4.3 渲染方案
- 游戏区240×240, 信息栏80×240
- 瓦片尺寸动态: 格子≤10用22px, 格子>10用16px
- 脏矩形: 每次移动只重绘2-4个格子
- 全部图形程序化绘制, 零图片资源

### 4.4 输入系统
- 按键: 10ms轮询 + 去抖 + 长短按识别
- 短按(<0.8s): 方向移动
- 长按(>0.8s): 撤销
- 超长按(>2s, 仅右键): 返回菜单
- 串口: USART1中断接收, WASD=移动, U=撤销, R=重置, M=菜单

### 4.5 音效系统
- PC0蜂鸣器, 软件PWM
- 6种音效: 移动/推箱/撞墙/过关/选择/撤销
- 频率200-1047Hz, 时长25-150ms
